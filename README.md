# Travel Planner

An AI-powered travel guide generator. Fill out a form with your destination and trip details and get a personalized travel report — safety assessment, attractions, local cuisine, and practical info — generated by Claude in seconds.

## Features

- **Safety assessment** — RED / ORANGE / GREEN risk levels with specific warnings
- **Attractions** — 8–12 picks tailored to your interests (nature, food, art, adventure, etc.)
- **Cuisine guide** — must-try dishes, dining customs, tipping norms, dietary accommodations
- **Practical info** — currency, transport, electricity, language phrases, visa requirements, emergency contacts, weather & packing tips

## Tech Stack

- **Next.js 16** (App Router) + TypeScript
- **Tailwind CSS v4**
- **Anthropic SDK** — two-model AI pipeline (Haiku + Sonnet)
- **Zod** + **react-hook-form** for validation

## Getting Started

### Prerequisites

- Node.js 18+
- An [Anthropic API key](https://console.anthropic.com)

### Setup

```bash
git clone <repo-url>
cd travel-planner
npm install
```

Create `.env.local`:

```
ANTHROPIC_API_KEY=sk-ant-...
```

Start the dev server:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## How It Works

The app uses a two-stage AI pipeline on every request:

```
User Form
   |
   v
Stage 1 — Haiku (claude-haiku-4-5-20251001)
   Validates destination, extracts trip profile
   temperature=0, max_tokens=500
   |
   | (if invalid → 422 INVALID_DESTINATION)
   v
Stage 2 — Sonnet (claude-sonnet-4-6)
   Generates full travel report as JSON
   temperature=0.7, max_tokens=16000
   |
   v
TravelReport displayed in UI
```

Haiku runs first because it's fast and cheap. If the destination isn't recognized, Sonnet is never called. Sonnet receives Haiku's extracted trip profile as context to produce a more personalized report.

## API

### `POST /api/travel`

**Request body** (`TravelFormData`):

| Field | Type | Description |
|---|---|---|
| `destination` | `string` | City or country name |
| `departureDate` | `string` | ISO date (YYYY-MM-DD) |
| `returnDate` | `string` | ISO date, must be after departure |
| `accommodation.booked` | `boolean` | Whether accommodation is already booked |
| `accommodation.preferences` | `AccommodationType[]` | hotel, airbnb, hostel, resort, b&b |
| `group.adults` | `number` | 1–20 |
| `group.children` | `number` | 0–20 |
| `group.type` | `GroupType` | Solo, Couple, Family, Friends, Business |
| `preferences` | `TravelPreference[]` | At least one required |

**Success response** (`200`):

```json
{ "success": true, "report": { ...TravelReport } }
```

**Error responses**:

| Status | Code | Meaning |
|---|---|---|
| `400` | `VALIDATION_ERROR` | Invalid request body |
| `422` | `INVALID_DESTINATION` | Destination not recognized |
| `429` | `RATE_LIMIT` | Anthropic rate limit hit |
| `500` | `AI_ERROR` | Unexpected model error |
